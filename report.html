Graph Coloring — University Exam Scheduling

1. Problem Overview

A university must schedule final exams so that no student sits two exams at the same time. If two courses share at least one student, they conflict and cannot be assigned the same time slot.

This is modeled as a Graph Coloring problem:

  • Vertices — Courses
  • Edges — Conflicts (shared students between two courses)
  • Colors — Exam time slots

The goal is to assign the minimum number of colors (time slots) such that no two adjacent vertices (conflicting courses) share the same color.

4. Algorithm — Backtracking Graph Coloring

The program uses a Backtracking algorithm to assign time slots. It tries each color for the current course, checks safety, recurses, and backtracks if no valid assignment is found.

Core Backtracking Logic

def _backtrack(graph, num_colors, colors_assigned, course, deadline):
    if course == len(graph):
        return True          # All courses colored — solution found

    if time.time() > deadline:
        return False         # Time limit reached

    for color in range(1, num_colors + 1):
        if is_safe(graph, course, color, colors_assigned):
            colors_assigned[course] = color
            if _backtrack(graph, num_colors, colors_assigned, course + 1, deadline):
                return True
            colors_assigned[course] = 0   # Backtrack

    return False

Safety Check

def is_safe(graph, course, color, colors_assigned):
    for neighbor in range(len(graph)):
        if graph[course][neighbor] == 1 and colors_assigned[neighbor] == color:
            return False
    return True

Strategy

  1. Run Greedy Coloring first — gives a valid solution instantly as an upper bound.
  2. Run Backtracking to try reducing by 1 color (with a 5-second time limit).
  3. Return the best solution found.

6. Program Structure

FunctionDescription
create_graph() Generates a 50×50 adjacency matrix with realistic random conflicts
is_safe() Checks if assigning a color to a course causes any conflict with neighbors
_backtrack() Recursive backtracking core — tries colors, recurses, backtracks on failure

display_result() Prints the final exam schedule grouped by time slot
conflict_report() Verifies the solution — confirms no two conflicting courses share a slot

7. Complexity Analysis

Worst Case Time Complexity: O(m ^ n)

  m = number of colors (time slots)
  n = number of vertices (courses)

Example: m=5, n=50 → 5^50 ≈ 10^34 operations (theoretical max)

Backtracking optimization:
  · Prunes branches the moment a conflict is detected
  · Avoids exploring the full exponential search tree
  · Greedy upper bound reduces the search range significantly
  · Time limit prevents worst-case hangs in practice
